# El Pulsar - Plataforma Descentralizada

**Live URL**: https://elpulsar.app  
**Documentación**: https://docs.elpulsar.app

## Estructura del Proyecto

Este proyecto ahora tiene una estructura optimizada que separa el sitio principal de la documentación:

```
el-pulsar/
├── index.html           # Sitio principal (elpulsar.app)
├── docs/
│   └── index.html       # Documentación (docs.elpulsar.app)
├── vercel.json          # Configuración de rutas Vercel
├── CNAME
└── README.md
```

## Despliegue

### Sitio Principal (elpulsar.app)
- **Plataforma**: Vercel
- **Tecnología**: HTML5, CSS3, JavaScript
- **Dominio**: elpulsar.app

### Documentación (docs.elpulsar.app)
- **Plataforma**: GitHub Pages
- **Carpeta**: `/docs`
- **Dominio**: docs.elpulsar.app

## Componentes del Ecosistema

### Core Components
- **DSL Lacho**: Lenguaje de dominio específico para contratos inteligentes
- **Devibe**: Sistema de resonancia energética y matching
- **Dirime**: Resolución de disputas descentralizada
- **EPIP Token**: Token nativo del ecosistema
- **Agents**: Agentes de IA para automatización
- **JSON-RPC API**: Interfaz de programación

### Tecnología
- **Frontend**: HTML5, CSS3, JavaScript (Responsive Design)
- **Backend**: Node.js, Python
- **Base de Datos**: Supabase (PostgreSQL)
- **Despliegue**: Vercel, GitHub Actions, GitHub Pages

## Filosofía Island Park

El Pulsar sigue la filosofía Island Park de transición orgánica desde economía descentralizada hacia economía formal:

- **Descentralización Inicial**: Estructuras distribuidas y flexibles
- **Transición Orgánica**: Evolución natural hacia mayor formalización
- **Participación Comunitaria**: Involucramiento de todos los stakeholders
- **Sostenibilidad**: Viabilidad económica y ambiental a largo plazo
- **Innovación Continua**: Adaptación y mejora constante

### GitHub Pages Configuration
- **Source**: Branch `main`

## Desarrollo

### Para Contribuir
1. Clona el repositorio: `git clone https://github.com/tu-usuario/el-pulsar.git`
2. Crea una rama para tu feature: `git checkout -b feature/nueva-funcionalidad`
3. Realiza tus cambios y commitea: `git commit -m 'Add nueva funcionalidad'`
4. Push a la rama: `git push origin feature/nueva-funcionalidad`
5. Abre un Pull Request

### Estructura de Archivos
- `index.html` - Página principal del sitio
- `docs/index.html` - Documentación técnica
- `vercel.json` - Configuración de Vercel
- `CNAME` - Configuración de dominios
